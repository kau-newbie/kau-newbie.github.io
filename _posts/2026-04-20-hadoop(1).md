**서론**

hadoop은

- HDFS
- MapReduce

로 이루어져있다. 모두 대규모 분산 시스템을 이룬다. 즉, 정말 많은 수의 노드(이하 '서버'와 혼용.)가 서로 Network(이하 NW)로 이루어져 하나의 거대한 시스템을 이루고 있다는 뜻이다.

여기서 둘은 모두, 중앙에서 통제하는 master node들과, 통제에 따라 일을 수행하는 slave nodes로 이루어져있다.
- 요즘은 용어가 좀 그래서 leader - follower node라고 부르기도 한댄다.
- 엄밀히는 각각 Namenode - Datanode, Jobtracker - Tasktracker 라고 부른다.
    - job은 유저가 요청한 일의 단위이다. 여러 task(더 작게 나눈 일의 단위)로 나눌 수 있다. 이 말인 즉슨, 일련의 여러 과정들을 포함할 정도로 큰 작업이다.
- 보통 `namenode`와 `jobtracker` 이 2개를 같이 하나의 노드에 넣어둔다. 하나의 노드에서 돌아가게 한다.

하둡이 좋은 이유는, 아래에서 반복하겠지만, 우리가 신경써서 정해야할(설계 및 프로그래밍) 부분을 매우매우 줄여준다는 것이다.

# Hadoop 공부 정리

배운 바로는, 빅데이터 처리 과정은 다음과 같다.

1. 정말 많은 수의 records(저장된 데이터)들에 대해 반복 작업을 수행한다.
2. **records들에게서 우리가 관심 있는 부분만 추출한다.** `map`
3. 분산 노드들 사이에서 중간 결과들이 정렬되고, 이동한다.
4. **중간 결과를 합쳐준다.** `reduce`
5. 최종 결과물을 출력한다.

여기서 우리는 2번과 4번, 즉 각각 "map"과 "reduce"만 구현해준다. 

그 외 나머지는 하둡이라는 runtime framework가 해줄 것이다.

## Map-Reduce 과정

하둡에서 개발자인 우리는 기본적으로 map()과 reduce()만 정의하면 된다고 한다.

### Map 과정

1. 사용자의 인풋은 master node로 가고, 다시 master node의 지시가 mapper에게 전달된다. 
- mapper는 map()을 수행하도록 master node에게 명령 받은 노드이다.
- map 프로그램 자체가 데이터가 있는 노드로 간다.
- 여기서 저장된 데이터 파일은 'split'이라는 단위로 읽힌다.
2. mapper에서는
    1. **map**을 수행한다. 의미 있는 key값과 그에 따른 value로 (<k,v>) domain(데이터 형태)이/가 변경된다.
    2. map()과정의 중간 결과물들은 mapper 노드의 local buffer에 차곡차곡 저장된다.
    3. buffer의 80% 정도가 차면, spill.out 이라는 형태의 파일로 디스크에 저장한다.
        - 디스크로 보내기 전, 버퍼에서는 `partition`과 `local sort`를 수행한다.
        - partition: 중간 결과를 몇 개로 쪼갤 것이냐? 
            - 보통 reducer의 갯수만큼 쪼갠다. 설정을 따로 할 수 있다.
            - partition들은 하나의 spill파일 안에 들어간다. 나중에 reducer의 요청에 해당하는 spill 파일 일부(특정 partition!)를 읽어 보내준다.
        - local sort: 같은 파티션 안에는 여러 개의 key-value 쌍들이 있다. 이 <k,[v]>들을 정렬해준다.
    4. combiner를 미리 설정해놓았으면, spill 파일들을 정렬한 후, 합쳐 하나의 map output file로 출력한다.
    5. reducer의 요청에 따라 **shuffle**(network를 타고 partition을 지정된 reducer node로 보내기)
    - 이때! shuffle은 각 map output file이 완성될 때마다 개별적으로 수행하게 된다.
    - 모든 input을 처리할 때까지 (shuffle을 하지 않고) 기다리는 건, reducer를 기다리게 만드는 일이므로 낭비이기 때문이다.

### Reduce 과정

### 간단한 예시 - word count

- String docid: document.txt 파일의 id
- String text: 실제 txt 파일의 내용

```java
Map(String docid, String text):
    for each word w in text:
        Emit(w,1); // reducer에게 보낸다.
        /**
         * 예시:
         * Map()을 거치면,
         *  ("apple",1), ("apple",1), ..., ("love",1)... 와 같은 output 생성
         * 
         * 이제 shuffle&sort 단계에서
         *  - 위에서 배운 buffer(partitioning & local sort) -> local disk -> combined map output file
         *  - local sorting을 거친 뒤, 새로운 domain을 가지게된 <k,v> 데이터는 ("apple",[1,1,1,1],... ) 형태일 것이다.
         * 마지막으로 shuffle로 각 partition 단위로 읽어 reducer에게 보낸다.  
         * 
         * Rduce()의 인자로,
         * - term = (sorted [<k,[v]>]).k
         * - values = (sorted [<k,[v]>]).v
         * 이렇게 넘겨준다.
         **/

Reduce(String term, Iterator<Int> values):
    int sum=0;
    for each v i nvalues:
        sum += v;
        Emit(term, sum);

        // 최종적으로, ("apple", 4), .... 로 결과가 나올 것이다.
```
