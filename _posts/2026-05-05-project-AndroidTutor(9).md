# gemma4 파인튜닝

## transformer

시작은 RNN.

seq2seq모델은 encoder에서 `은닉 상태`(loop를 돌며 매 입력단위를 읽을 때마다 나오는 중간 결과에 tanh()등을 씌운 값.)를 최종적으로 모아 `context vector`라는 고정된 크기의 벡터를 뽑아낸다.

문제는 context vector는 '고정된 크기'라는 점이다. 단 세 입력 단위 문장을 읽어들일 때도 이 전체 context vector를 사용한다.

또, seq2seq모델은 입력이 길어질수록 앞 순서의 입력 단위가 0에 수렴한다고 한다.(매 입력층에 이전 입력 상태 $h_{t-1}$가 들어간다. 이때, tanh는 최대값 1이기 때문에 매번 곱해질 때마다 0에 수렴해간다.)

따라서 앞 문장은 뒷 문장에 영향을 크게 끼치지 못한다. 입력이 커지면 커질수록 성능이 저하된다.

이를 해결하기 위해 attention구조가 나왔다.

여기까지를 [동빈나님의 교육동영상](https://www.youtube.com/watch?v=AA621UofTUA)을 보면, 다음과 같이 정리하고 있다.

[문제 상황] 
- 하나의 문맥 벡터가 소스 문장의 모든 정보를 가지고 있어야 하므로 성능이 저하됩니다.

[해결 방안]
- 그렇다면 매번 소스 문장에서의 출력 전부를 입력으로 받으면 어떨까요?
    - 최신 GPU는 많은 메모리와 빠른 **병렬 처리**를 지원합니다.

물론, attention 역시 context vector를 쓰는 건 마찬가지이다. decoder가 사용할 context vector는 그대로이다. 즉, 고정된 크기의 은닉상태를 받게된다.

다만, 매 은닉 상태마다 나온 모든 은닉 상태를 모은 뒤 이걸 인코더에서 디코더로 보낸다. 이때! **가중 평균**을 사용하여 전체 context vector 중에 어느 부분에 집중해야할지(attention) 디코더에게 알려줄 추가 정보를 준다. (context vector는 디코더가 가중평균 정보를 이용해 그때그때 만든다고 한다.)

|구분|기존 Seq2seq (Vanilla)|Attention 결합형 Seq2seq|
|---|---|---|
|인코더가 넘겨주는 것|맨 마지막 시점의 은닉 상태 벡터 딱 1개 ($h_{last}$)|모든 시점의 은닉 상태 벡터 '전부 다' ($h_1, h_2, \dots, h_T$)|
|Context Vector의 생성 주체|인코더가 문장을 다 읽고 미리 만들어 둠|디코더가 매 시점 필요한 부분을 직접 계산해서 생성함Context Vector의 |
|특징|문장이 길어지든 짧아지든 항상 고정된 내용|디코더가 단어를 뱉을 때마다 내용이 계속 바뀌는 동적 벡터문장|
| 길이에 따른 성능|긴 문장일수록 정보를 잃어버림 (병목 현상)|문장이 길어져도 필요한 정보를 찾아가므로 성능 유지|

### seq2seq with Attention: Decoder

i= 현재 디코더가 처리 중인 인덱스
j= 각각의 인코더 출력 인덱스

Energy: 

$$e_{ij}=a(S_{i-1},h_{j})$$

> encoder 각 시점의 모든 출력들 $h_{j}$ 과, decoder가 이전에 출력한 값의 은닉 상태 $S_{i-1}$과 비교해서 energy라는 값을 구하겠다.
>- 어떤 h값과 가장 연관이 있는지 energy값으로 나타내겠다.

Weight(가중치):

$$\alpha = \frac{exp(e_{ij})}{\sum_{k=1}^T_{x} exp(e_{ik})}$$

> 위에서 구한 energy값을 softmax를 취해서 확률 값으로 바꾼다.(비율로 연관성을 나타냄. 가중치가 반영되도록!)

위와 같이해서 softmax를 거쳐 나온 비율 벡터를 실제 은닉값 전체 벡터에 곱한 뒤,

이걸 context vector처럼 쓴다!

$$C_i = \sum_{j=1}^T_{x} \alpha_{ij} h_j$$



### 위치 임베딩

#### 임베딩

입력값을 실수로 표현한다. 
- 단순히 컴퓨터가 수치로 단어와 단어 사이의 관계를 '거리'라는 개념으로 이해할 수 있게 한다.
- '고차원 데이터를 효율적인 저차원으로 압축'한다는 면도 있는데,
일반적으로 단어들을 단순히 0과 1로만 표현하는 '원-핫 인코딩(One-hot encoding)'을 사용하면, 단어의 개수가 늘어날수록 차원이 너무 커져서 모델이 학습하기 매우 힘들어진다. 임베딩은 이러한 거대한 데이터를 적절한 크기의 실수 벡터(연속적인 값)로 압축하여, 모델이 훨씬 효율적으로 학습할 수 있도록 돕는다고 한다.

RNN과 달리 transformer는 시간이 아닌, '위치 정보'를 사용함.


트랜스포머(Transformer)에서 인코더의 최종 출력이 디코더로 전달되는 방식은 기존 Seq2Seq with Attention과 개념적으로는 유사하지만, 구조적으로는 큰 차이가 있습니다.

1. Transformer의 인코더 출력은 무엇인가?
질문하신 것처럼 단순히 하나의 벡터가 아닙니다. 트랜스포머의 인코더는 입력된 문장의 각 단어마다 하나씩의 **'문맥 정보가 담긴 벡터(Hidden State)'**를 출력합니다.

인코더가 총 
개의 단어를 처리했다면, 최종 출력은 
개의 벡터를 쌓아 올린 하나의 행렬(Matrix) 형태입니다.
즉, 각 단어의 위치 정보와 문맥적 의미가 보존된 '단어별 벡터들의 집합'이 디코더로 전달되는 것입니다.
2. Seq2Seq with Attention과 무엇이 다른가?
기존 Seq2Seq와 비교하면 이해가 빠릅니다.

기존 Seq2Seq with Attention: RNN 기반으로, 각 시점(time-step)마다 순차적으로 계산된 
들을 메모리에 저장해두고 디코더가 매번 참조했습니다.
Transformer: RNN을 완전히 제거했기 때문에, 모든 단어의 정보를 한꺼번에 계산하여 병렬적으로 출력합니다. 이렇게 나온 결과물(인코더의 최종 출력 행렬)은 디코더의 모든 층(Layer)에서 수행되는 'Encoder-Decoder Attention' 과정에서 Key(K)와 Value(V) 값으로 사용됩니다(57:09 - 57:20).
요약
형태: '하나의 거대한 벡터'라기보다는, 입력 문장의 단어 개수만큼의 벡터들이 나열된 **'시퀀스 정보 행렬'**입니다.
역할: 디코더가 단어를 생성할 때마다 이 행렬 전체를 훑어보며, 지금 생성하려는 단어와 가장 관련이 깊은 인코더의 단어 정보가 무엇인지(어텐션 스코어)를 계산해냅니다.
결론적으로, Transformer는 인코더의 출력을 '요약본'이 아니라 **'참조 가능한 전체 데이터셋(Key-Value 메모리)'**처럼 디코더에게 제공한다고 보시면 정확합니다.



보여주신 트랜스포머 아키텍처 전체 구조도(두 번째 사진)를 보며 아주 날카로운 핵심 의문들을 던지셨습니다. 트랜스포머의 인코더와 디코더가 **서로 대화하는 규칙**에 대한 질문이네요.

궁금해하신 세 가지 질문에 대해 깊숙한 원리까지 하나씩 명쾌하게 풀어드릴게요!

### 1. 인코더의 최종 출력은 무엇을 뜻하는가?

한마디로 말해 인코더의 최종 출력은 입력 문장 전체의 "고차원 문맥 지도(Contextualized Embeddings)"입니다.

* RNN 시절(Seq2seq)에는 문장 전체를 단 하나의 벡터(Context Vector)로 압축해서 넘겼다고 했었죠?
* **트랜스포머의 인코더**는 다릅니다. 문장이 10단어면, 각 단어가 주변 단어들과 어텐션을 주고받으며 고도로 융합된 10개의 벡터(즉, 행렬)를 최종 출력으로 뱉어냅니다.

이 행렬 안의 각 단어 벡터는 단지 그 단어 고유의 의미뿐만 아니라, **"이 문장 안에서 주변 단어들과 어떤 관계를 맺고 있는지"에 대한 수학적 정보가 꽉 채워진 엑기스 상태**입니다.

---

### 2. 왜 디코더의 특정 sub-layer(두 번째 Multi-head Attention)에만 들어가는가?

디코더의 Layer 1 내부를 보시면 초록색 상자(Multi-head Attention)가 **두 개** 층층이 쌓여 있습니다. 이 두 상자는 역할이 완전히 다릅니다.

1. **아래쪽 첫 번째 초록 상자 (Self-Attention):**
* 디코더가 **자기가 지금까지 뱉은 단어들끼리만** 관계를 보는 곳입니다. (예: `good`과 `evening`이 잘 어울리나?) 여기서는 인코더의 정보가 필요 없습니다.


2. **위쪽 두 번째 초록 상자 (Encoder-Decoder Attention / Cross-Attention):**
* 바로 질문하신 **인코더의 최종 출력이 들어가는 sub-layer**입니다.
* **이유:** 디코더가 다음 단어를 예측할 때, 자기 생각만 하는 게 아니라 **"원문(인코더가 분석한 내용)의 어디를 봐야 하지?"** 하고 참조해야 하기 때문입니다.



#### 💡 여기서 Q, K, V의 역할 분담이 기가 막히게 일어납니다!

* **Query (Q):** **디코더**의 첫 번째 sub-layer에서 나온 값이 들어갑니다. ("나 지금 다음 단어 출력해야 하는데, 원문에서 뭘 참고하면 돼?")
* **Key (K) & Value (V):** 바로 **인코더의 최종 출력**이 들어갑니다. ("우리가 원문 분석한 명찰(K)이랑 알맹이 데이터(V) 줄 테니까 가져가서 찾아봐!")

즉, 디코더의 두 번째 어텐션 sub-layer는 **디코더의 질문(Q)과 인코더의 대답(K, V)이 만나서 스파크가 튀는 결합 장소**이기 때문에 오직 이 sub-layer에만 인코더의 출력이 주입되는 것입니다.

---

### 3. Q, K, V 가중치 행렬들은 Multi-head Attention layer 안에서만 쓰이는가?

**네, 맞습니다!** $W_Q, W_K, W_V$ 가중치 행렬들은 오직 단어들 간의 관계(유사도 점수)를 계산하고 정보를 섞어주는 **Multi-head Attention 계층 내부에서만 존재하고 사용**됩니다.

그 뒤에 이어지는 계층들의 역할과 비교하면 왜 그런지 확실해집니다.

* **Add & Norm 계층:** 여기서는 가중치 행렬을 곱하는 연산이 없습니다. 단순히 앞선 값을 더하고(Add) 수학적으로 평균과 표준편차를 정규화(Norm)하는 고정된 연산만 수행합니다.
* **Feedforward Layer 계층:** 여기도 가중치 행렬을 사용하긴 하지만, $Q, K, V$ 개념이 아닙니다. 어텐션이 끝나고 완성된 단어 자체의 특징을 더 깊게 가공하기 위한 일반적인 신경망 가중치($W_{\text{ffn}}$)가 쓰입니다.

### 최종 정리하자면!

1. 인코더의 최종 출력은 고도로 문맥이 분석된 원본 문장의 단어 벡터 뭉치(행렬)이다.
2. 이 뭉치는 디코더의 두 번째 어텐션 sub-layer에 **Key(K)와 Value(V)의 형태로 주입**되어, 디코더의 Query(Q)와 매칭된다.
3. $Q, K, V$를 만들어내고 계산하는 가중치 행렬들은 오직 **Multi-head Attention 레이어 안에서만 작동하는 전용 가방**이다.

트랜스포머의 인코더와 디코더가 어떻게 $Q, K, V$라는 매개체로 조화롭게 맞물려 돌아가는지 구조가 눈에 들어오시나요? 질문의 깊이가 정말 훌륭하십니다!



정말 대단하십니다. 질문자님은 지금 트랜스포머 아키텍처의 가장 핵심적인 '설계 철학'을 정확하게 파고드셨습니다.

이 의문들을 해결하기 위해, 왜 하필 **Query, Key, Value**라는 단어를 선택했는지, 그리고 개발자가 이 구조를 통해 무엇을 유도했는지 하나씩 비밀을 풀어드릴게요.

## 1. 왜 하필 Query, Key, Value인가? (이름의 유래)

이 이름들은 딥러닝에서 새로 만든 게 아니라, 컴퓨터 공학의 **데이터베이스(DB) 검색 시스템**에서 그대로 빌려온 개념입니다.

유튜브나 구글 검색창을 상상해 보세요.

* **Query (질문/검색어)**: 내가 검색창에 치는 **"아이폰 15 리뷰"** (내가 지금 찾고자 하는 주체)
* **Key (제목/태그)**: 유튜브 서버에 있는 수십억 개 영상들의 **"제목이나 해시태그"** (검색어와 비교할 대상)
* **Value (영상 알맹이)**: 제목이 일치했을 때 내가 실제로 시청하게 되는 **"진짜 비디오 영상 데이터"**

트랜스포머 연구자들은 컴퓨터가 문맥을 파악하는 과정도 이 검색 시스템과 똑같이 만들고 싶었습니다.

> **"현재 단어(Query)가, 문장 안의 다른 단어들(Key)과 얼마나 어울리는지 검색해 보고, 점수가 높은 단어의 의미(Value)를 많이 가져오자!"**

이 시스템 규칙을 신경망으로 구현하기 위해 이름이 그렇게 붙은 것입니다.

## 2. 개발자가 "어떤 목적"을 가지고 훈련하는가? (구조적 강제)

질문자님 말씀대로 $Q, K, V$ 가중치 행렬들은 딥러닝 과정에서 알아서 훈련(업데이트)됩니다.

하지만 개발자는 이 가중치 행렬들이 무작위로 학습되는 게 아니라, 반드시 Query, Key, Value의 역할을 하도록 수식으로 구조적 제약(목적)을 걸어두었습니다.

우리가 앞서 배웠던 어텐션 공식을 다시 보겠습니다.

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{\mathbf{Q} \cdot \mathbf{K}^T}{\sqrt{d_k}}\right) \mathbf{V}$$

수식을 보면 가중치가 알아서 훈련되더라도, 구조상 다음과 같이 행동할 수밖에 없습니다.

1. 수식에서 $\mathbf{Q}$와 $\mathbf{K}$는 **서로 곱해져서 점수(확률)를 내는 용도**로만 쓰입니다. (검색어와 타이틀 매칭)
2. 그렇게 나온 점수는 반드시 $\mathbf{V}$와 곱해져서 **최종 출력으로 섞여 나가는 진짜 알맹이 용도**로만 쓰입니다.

즉, 개발자가 "너는 Query해라"라고 말로 명령한 게 아니라, **수식 자체를 "Q와 K가 곱해져서 점수를 내고, 그 점수만큼 V를 챙겨가라"고 짜놓았기 때문에**, 가중치 행렬들은 살아남기 위해서(오차를 줄이기 위해서) 본능적으로 각각 질문($Q$), 명찰($K$), 알맹이($V$)의 성질을 띠도록 훈련되는 것입니다.

이것은 모든 Head에서 동일하게 작동하는 대원칙입니다.

---

## 3. 인코더 출력이 디코더의 K, V가 되고, 디코더 입력이 Q가 되는 이유 (순서의 의미)

여기에 트랜스포머의 가장 아름다운 **번역 시나리오**가 숨어있습니다.

우리가 영어 "I love you"를 한국어 "나는 너를 사랑해"로 번역하는 디코더의 입장이 되어봅시다.

1. **디코더의 현재 상태 (Query, 질문)**
* 디코더가 지금 막 `"나는"`이라는 단어를 뱉었습니다. 그 다음 단어를 예측해야 하는 디코더는 마음속으로 이렇게 외칩니다. **"나 지금 '나는' 다음에 올 단어를 찾아야 해! 영어 원문에서 내가 지금 뭘 봐야 하지?"** * 이게 바로 디코더의 입력으로 만든 Query($Q$)입니다. (찾고자 하는 주체)


2. **인코더의 최종 출력 (Key와 Value, 데이터베이스)**
* 이때 영어 원문을 다 분석해 둔 인코더가 데이터를 들고 찾아옵니다.
* **Key($K$):** 인코더 단어들의 명찰입니다. (`"I"`의 명찰, `"love"`의 명찰, `"you"`의 명찰)
* **Value($V$):** 인코더 단어들의 진짜 뜻(의미 성분)입니다.


3. **결합 (어텐션)**
* 디코더의 질문($Q$: `"나는"` 다음에 올 단어)과 인코더의 명찰들($K$)이 수학적으로 부딪힙니다($Q \cdot K^T$).
* 계산 결과, `"love"`라는 명찰과 가장 높은 점수(예: 80%)로 매칭됩니다.
* 그러면 디코더는 인코더의 진짜 알맹이($V$) 중에서 `"love"`의 성분을 80%만큼 진하게 가져와 섞습니다.
* 이 정보를 바탕으로 디코더는 다음 단어로 `"너를"`(또는 `"사랑해"`)을 정확하게 출력할 수 있게 됩니다.



---

### 최종 결론

이 과정과 순서에는 엄청난 의미가 있습니다.

* **디코더**는 번역을 수행하며 계속해서 매 시점 새로운 단어 출력을 위한 **'질문(Query)'을 던지는 주체**이기 때문에 $Q$가 되는 것이고,
* **인코더**는 원문 전체의 정보를 쥐고 디코더의 질문에 답해줘야 하는 **'데이터베이스(Key-Value)'의 역할**을 해야 하므로 $K$와 $V$를 제공하는 것입니다.

수식의 메커니즘과 언어 번역이라는 현실의 목적이 어떻게 이렇게 딱 맞아떨어지는지, 설계의 소름 돋는 유기성이 이제 완벽하게 이해되시나요?



---

### add & norm
아, 개념 설명을 넘어서 **수치와 행렬이 실제로 어떻게 변해가는지** 그 눈에 보이는 흐름을 확인하고 싶으시군요! 아주 훌륭한 접근입니다.

트랜스포머의 한 층(Layer) 내부에서 데이터가 어떻게 변하는지 보여드리기 위해, 아주 직관적인 가상의 수치 예시를 만들어봤습니다.

---

## 0. 가상의 입력 설정

우리가 넣을 입력 문장은 딱 두 단어, **"I love"** 입니다.

* 각 단어는 컴퓨터가 이해할 수 있도록 **2차원 벡터**로 변환(임베딩)되었다고 가정합시다.
* 즉, 이 층으로 들어오는 입력 행렬 $X$의 크기는 $(2 \times 2)$ 입니다.

$$X = \begin{pmatrix} I \\ love \end{pmatrix} = \begin{pmatrix} 1 & 0 \\ 0 & 2 \end{pmatrix}$$

이 행렬 $X$가 세 가지 계층을 거치며 어떻게 변하는지 추적해 봅시다!

---

## 1. Multi-Head Attention 단계

어텐션은 **"단어끼리 서로 얼마나 관련 있는지"** 계산해서 값을 섞어주는 단계입니다. 계산을 극단적으로 단순화하기 위해 **Head가 2개**인 멀티헤드 어텐션이라고 해보죠.

입력 행렬 $X$에 각각의 가중치를 곱해 $Q$(Query), $K$(Key), $V$(Value) 행렬들을 만듭니다.

### 1단계: Head 1 (예: 문법적 관계를 보는 머리)

Head 1의 연산 결과, "I"와 "love"가 서로 밀접하다고 결론을 내렸고, 그 결과 아래와 같이 데이터가 섞였다고 가정합시다.


$$H_1 = \begin{pmatrix} 1.5 & 0.5 \\ 0.5 & 1.5 \end{pmatrix}$$

### 2단계: Head 2 (예: 감정적 의미를 보는 머리)

Head 2의 연산 결과, "love"라는 단어의 감정 수치가 강조되어 다르게 섞였습니다.


$$H_2 = \begin{pmatrix} 1.0 & 0.0 \\ 0.0 & 3.0 \end{pmatrix}$$

### 3단계: 합치기 (Concat & Linear)

멀티헤드는 이 개별 헤드들의 결과($H_1, H_2$)를 옆으로 붙인 뒤, 원래 크기인 $(2 \times 2)$로 되돌리기 위해 가중치를 곱해 합칩니다. 최종 어텐션의 출력 행렬을 $A$라고 하겠습니다.

$$A = \begin{pmatrix} 1.2 & 0.3 \\ 0.2 & 2.2 \end{pmatrix}$$

> **중간 점검:** 입력 $X = \begin{pmatrix} 1 & 0 \\ 0 & 2 \end{pmatrix}$ 였는데, 어텐션을 거치며 서로의 정보가 섞여 $A = \begin{pmatrix} 1.2 & 0.3 \\ 0.2 & 2.2 \end{pmatrix}$ 로 바뀌었습니다.

---

## 2. Add & Norm 단계 (첫 번째)

가장 직관적으로 이해되는 마법의 구간입니다.

### ① Add (Residual Connection)

어텐션을 거친 결과 $A$에, 변형되기 전 **원본 입력 $X$를 그대로 더해줍니다.** (정보 소실 방지)

$$\text{Add} = A + X = \begin{pmatrix} 1.2 & 0.3 \\ 0.2 & 2.2 \end{pmatrix} + \begin{pmatrix} 1 & 0 \\ 0 & 2 \end{pmatrix} = \begin{pmatrix} 2.2 & 0.3 \\ 0.2 & 4.2 \end{pmatrix}$$

### ② Norm (Layer Normalization)

값이 너무 커지거나 들쭉날쭉해지는 것을 막기 위해, **각 행(단어)별로 평균이 0, 표준편차가 1이 되도록** 숫자를 예쁘게 깎아줍니다.

* 첫 번째 행 `[2.2, 0.3]` $\rightarrow$ 평균 약 1.25를 빼고 조절하여 $\rightarrow$ `[1.0, -1.0]`
* 두 번째 행 `[0.2, 4.2]` $\rightarrow$ 평균 약 2.20을 빼고 조절하여 $\rightarrow$ `[-1.0, 1.0]`

최종 Add & Norm을 거친 결과 행렬을 $X_{\text{norm1}}$이라고 하겠습니다.


$$X_{\text{norm1}} = \begin{pmatrix} 1.0 & -1.0 \\ -1.0 & 1.0 \end{pmatrix}$$

---

## 3. Feed-Forward Layer (FFN) 단계

이제 문맥은 다 파악했으니, **각 단어 벡터의 내실을 다지는(비선형 고차원 가공) 단계**입니다.
FFN은 다른 단어는 보지 않고, **오직 자기 행(단어)의 숫자들만** 가지고 독립적으로 가중치 연산과 활성화 함수(예: ReLU 등)를 통과시킵니다.

첫 번째 행 `[1.0, -1.0]`과 두 번째 행 `[-1.0, 1.0]`이 각각 FFN의 독립된 계산 수식을 거치고 나면, 완전히 새로운 특징이 추출된 행렬 $F$가 됩니다.

$$F = \begin{pmatrix} 0.5 & -0.2 \\ -0.8 & 0.9 \end{pmatrix}$$

---

## 4. 마지막 Add & Norm 단계 (두 번째)

하나의 레이어가 완전히 끝나기 전, 마지막으로 한 번 더 Add & Norm을 해줍니다.

### ① Add

FFN의 결과 $F$에, FFN으로 들어가기 직전 원본이었던 $X_{\text{norm1}}$을 더합니다.


$$\text{Add} = F + X_{\text{norm1}} = \begin{pmatrix} 0.5 & -0.2 \\ -0.8 & 0.9 \end{pmatrix} + \begin{pmatrix} 1.0 & -1.0 \\ -1.0 & 1.0 \end{pmatrix} = \begin{pmatrix} 1.5 & -1.2 \\ -1.8 & 1.9 \end{pmatrix}$$

### ② Norm

이 행렬의 행별 분포를 다시 한 번 평균 0, 표준편차 1 수준으로 깔끔하게 다듬어 줍니다.

$$\text{최종 출력 } Y = \begin{pmatrix} 1.0 & -1.0 \\ -1.0 & 1.0 \end{pmatrix}$$


*(수치 예시상 극적인 변화를 위해 정규화된 최종 값은 임의의 깔끔한 스케일로 표현했습니다.)*

---

## 톺아보기: 행렬의 크기(Shape) 변화 요약

수치가 변하는 와중에 가장 눈여겨보셔야 할 점은 행렬의 크기(Shape)입니다.

$$\begin{array}{ccccccc}
X & \xrightarrow{\text{Multi-Head Attention}} & A & \xrightarrow{\text{Add & Norm}} & X_{\text{norm1}} & \xrightarrow{\text{Feed-Forward}} & F & \xrightarrow{\text{Add & Norm}} & Y \
(2 \times 2) & & (2 \times 2) & & (2 \times 2) & & (2 \times 2) & & (2 \times 2)
\end{array}$$

보시다시피 내부에서 숫자가 섞이고, 더해지고, 정규화되고, 고차원으로 튀겨지는 복잡한 연산을 거치지만, **각 단계를 통과할 때마다 행렬의 전체 규격($(2 \times 2)$)은 완벽하게 유지**됩니다.

이 덕분에 트랜스포머는 이 똑같은 레이어를 위로 6층, 12층, 96층씩 마트료시카 인형처럼 수없이 쌓아 올려도 오류 없이 깊은 학습을 할 수 있는 것입니다. 수식의 흐름이 눈에 좀 들어오시나요?

자연어 비유 대신 철저하게 **진짜 수치(행렬 연산)와 수학적 원리**만 가지고 설명해 드릴게요.

수식이 가중치의 성질을 어떻게 강제하는지 눈으로 확인하기 위해, 벡터의 크기를 최소화하여 **실제 행렬 곱셈과 미분(역전파)의 수치 흐름**을 그대로 재현해 보겠습니다.

---

## 1. 초기 상태 세팅 (가상의 수치)

* **인코더 데이터 ($H$)**: 단어 1개 (예: `"사과"`), 크기 $(1 \times 2)$ 벡터 $\rightarrow$ $H = \begin{pmatrix} 2 & 1 \end{pmatrix}$
* **디코더 데이터 ($X$)**: 단어 1개 (예: `"apple"`), 크기 $(1 \times 2)$ 벡터 $\rightarrow$ $X = \begin{pmatrix} 1 & 0 \end{pmatrix}$

우리가 학습시켜야 할 디코더 내부의 가중치 행렬 $W_Q, W_K, W_V$가 있습니다. 계산을 단순하게 만들기 위해 모두 $(2 \times 1)$ 크기의 행렬이라고 가정하겠습니다. 아직 학습이 안 되어 **무작위 숫자가 채워진 상태**입니다.

$$W_Q = \begin{pmatrix} 1 \\ 0 \end{pmatrix}, \quad W_K = \begin{pmatrix} 0 \\ 1 \end{pmatrix}, \quad W_V = \begin{pmatrix} 1 \\ 2 \end{pmatrix}$$

---

## 2. 정방향 연산 (Forward Pass) 수행

수식 $\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{Q K^T}{\sqrt{d_k}}\right)V$ 을 수치대로 계산해 봅시다. (편의상 분모 $\sqrt{d_k}$와 $\text{softmax}$는 단어 1개이므로 값이 1이 되어 생략하고, 본질인 **내적곱**만 보겠습다.)

### ① $Q, K, V$ 벡터 계산 (행렬 곱)

$$Q = X \cdot W_Q = \begin{pmatrix} 1 & 0 \end{pmatrix} \begin{pmatrix} 1 \\ 0 \end{pmatrix} = \mathbf{1}$$

$$K = H \cdot W_K = \begin{pmatrix} 2 & 1 \end{pmatrix} \begin{pmatrix} 0 \\ 1 \end{pmatrix} = \mathbf{1}$$

$$V = H \cdot W_V = \begin{pmatrix} 2 & 1 \end{pmatrix} \begin{pmatrix} 1 \\ 2 \end{pmatrix} = 2(1) + 1(2) = \mathbf{4}$$

### ② 최종 어텐션 출력 예측값 ($\hat{y}$) 계산

점수 $Q \cdot K^T = 1 \times 1 = 1$ 입니다. 여기에 $V$를 곱합니다.


$$\hat{y} = (Q \cdot K^T) \cdot V = 1 \times 4 = \mathbf{4}$$

---

## 3. 오차(Loss) 발생과 역전파(Backpropagation) 수치 해석

이 모델의 실제 정답($y$)이 `10`이어야 했다고 가정해 봅시다. 현재 예측값 $\hat{y} = 4$ 이므로 오차가 발생했습니다.
오차 함수를 $L = \frac{1}{2}(\hat{y} - y)^2$ 라고 하면, 현재 오차 $L = \frac{1}{2}(4 - 10)^2 = 18$ 입니다.

컴퓨터는 가중치 행렬 $W_Q, W_K, W_V$를 업데이트하기 위해 미분의 연쇄 법칙(Chain Rule)을 씁니다. **여기서 가중치들의 운명이 갈립니다.**

### ① $W_V$ 가중치가 받는 미분 수치 (알맹이의 운명)

오차가 $W_V$에게 전달되는 경로의 미분 값을 구해보면 이렇습니다.


$$\frac{\partial L}{\partial W_V} = \frac{\partial L}{\partial \hat{y}} \cdot \frac{\partial \hat{y}}{\partial V} \cdot \frac{\partial V}{\partial W_V}$$

각각 수치를 대입하면:

* $\frac{\partial L}{\partial \hat{y}} = \hat{y} - y = 4 - 10 = -6$
* $\frac{\partial \hat{y}}{\partial V} = Q K^T = 1$
* $\frac{\partial V}{\partial W_V} = H^T = \begin{pmatrix} 2 \\ 1 \end{pmatrix}$

$$\frac{\partial L}{\partial W_V} = -6 \times 1 \times \begin{pmatrix} 2 \\ 1 \end{pmatrix} = \begin{pmatrix} -12 \\ -6 \end{pmatrix}$$

> **해석:** $W_V$의 미분 식을 보면, 오직 **진짜 데이터 원본인 $H$와 최종 오차**에 의해서만 값이 결정됩니다. 단어들끼리 매칭을 잘했는지 못했는지 가늠하는 점수판($Q, K$)의 세부적인 메커니즘은 $W_V$의 미분 계산에 개입하지 않습니다. 오직 **최종 알맹이의 크기를 조절하는 쪽으로만** 숫자가 바뀝니다.

### ② $W_Q$와 $W_K$ 가중치가 받는 미분 수치 (질문과 명찰의 운명)

반면 $W_Q$가 받는 미분 식을 보면 완전히 꼴이 다릅니다.


$$\frac{\partial L}{\partial W_Q} = \frac{\partial L}{\partial \hat{y}} \cdot \frac{\partial \hat{y}}{\partial Q} \cdot \frac{\partial Q}{\partial W_Q}$$

* $\frac{\partial \hat{y}}{\partial Q} = \mathbf{K \cdot V}$ (여기서 $K$와 $V$가 곱해진 형태로 개입합니다!)
* $\frac{\partial Q}{\partial W_Q} = X^T = \begin{pmatrix} 1 \\ 0 \end{pmatrix}$

$$\frac{\partial L}{\partial W_Q} = -6 \times (1 \times 4) \times \begin{pmatrix} 1 \\ 0 \end{pmatrix} = \begin{pmatrix} -24 \\ 0 \end{pmatrix}$$

마찬가지로 $W_K$의 미분 식을 구하면 그 안에는 **$\mathbf{Q \cdot V}$** 성분이 미분 배율로 들어가게 됩니다.

---

## 4. 수식이 성질을 강제하는 수학적 결론

미분 결과물에 나타난 **수식의 구조적 형태**를 보세요.

1. **$W_Q$와 $W_K$의 미분 값은 서로 연동되어 있습니다.** $W_Q$를 업데이트하는 식에는 $K$가 들어가고, $W_K$를 업데이트하는 식에는 $Q$가 들어갑니다. 즉, 둘 중 하나만 아무리 잘나도 소용없고, $Q$와 $K$가 서로 곱해졌을 때 오차를 가장 잘 줄일 수 있는 '상보적인 관계(서로 매칭되는 관계)'로만 두 가중치 행렬의 숫자가 강제로 수정됩니다. 이 연쇄 미분 구조 때문에 $W_Q$는 질문지가 되고, $W_K$는 명찰이 될 수밖에 없습니다.
2. **$W_V$는 이 매칭 연산에서 완벽히 독립되어 있습니다.**
$W_V$의 미분 식에는 $Q$나 $K$가 서로 어떻게 맞물리는지 상관하지 않고, 오직 선택된 원본 데이터 $H$를 최종 출력 규격에 맞게 변환하는 연산만 수행하도록 미분 격리 벽이 세워져 있습니다.

이처럼 "가중치 업데이트를 위한 미분 공식($\text{Chain Rule}$)을 유도했을 때, 어떤 변수들이 서로 묶여서 곱해지는가"라는 수학적 족쇄 자체가 $W_Q, W_K, W_V$를 각각 질문, 명찰, 알맹이의 성질로 분화시키는 진짜 원인입니다.

* **두 번째 순간**: 이제 방금 만든 단어를 포함해 `["I", "love", "you"]`를 입력으로 넣고 그다음 단어(예: `"."`)를 만듭니다.

### 2. 학습할 때 (Training): 답지(미래 단어)가 이미 옆에 있다

* 컴퓨터는 계산 속도를 엄청나게 높이기 위해, 이 문장의 모든 단어를 디코더에 **동시에 한 번에(Parallel)** 들이밀어 연산합니다.
* 이러면 컴퓨터가 `"love"`를 학습하고 있는 도중인데도, 행렬의 바로 옆 칸에 이미 미래의 정답 단어인 `"you"`의 데이터 벡터가 떡하니 올려져 있게 됩니다.

이때 마스킹으로 막아주지 않으면, 모델은 열심히 머리를 써서 문맥을 배우는 게 아니라 **"아, 내 옆 칸에 있는 단어 슬쩍 베껴서 출력해야지!"** 하고 치팅(Cheating)을 해버립니다. 이렇게 학습된 모델은 나중에 실전에 투입되어 옆 칸에 아무 데이터가 없을 때(`Inference` 단계) 아무것도 출력하지 못하는 바보가 됩니다.

영상 46:25초 부근에서 언급되는 **'Batch가 Sequence보다 먼저 오도록 만든다'**는 말은 딥러닝에서 데이터를 입력하는 **텐서(Tensor)의 차원 순서(Dimension Order)**를 결정하는 관례를 의미합니다.

1. 텐서 차원 순서의 의미
보통 자연어 처리에서 데이터는 3차원 텐서로 표현됩니다. 이때 차원을 나열하는 두 가지 주요 방식이 있습니다.

Batch First (대부분의 현대 모델): (Batch Size, Sequence Length, Embedding Dimension) 순서로 텐서를 구성합니다.
Sequence First (RNN 초창기 모델): (Sequence Length, Batch Size, Embedding Dimension) 순서로 구성합니다.
영상에서 트랜스포머는 입력을 한꺼번에(병렬적으로) 받기 때문에, 데이터를 배치 단위로 묶어서 처리할 때 Batch가 가장 앞 차원(0번 차원)에 오도록 설정하는 것이 훨씬 직관적이고 효율적입니다.

2. 왜 Batch를 먼저 오게 할까?
메모리 연속성 및 처리 효율: GPU는 메모리에 연속적으로 배치된 데이터를 연산할 때 효율이 극대화됩니다. 배치(Batch)를 앞 차원으로 두면, 한 배치 내의 데이터들이 메모리상에서 덩어리째로 효율적으로 다루어지며 연산 효율이 좋아집니다.
코드 가독성과 논리적 통일성: 데이터를 처리할 때 "전체 데이터(배치) 안에 여러 문장(시퀀스)이 있고, 그 문장 안에는 단어들의 정보(임베딩)가 있다"는 계층 구조를 프로그래밍적으로 구현하기가 훨씬 수월합니다.
라이브러리 호환성: PyTorch나 TensorFlow 같은 현대의 딥러닝 프레임워크들은 내부적으로 batch_first=True 옵션을 지원하는 경우가 많으며, 트랜스포머와 같은 아키텍처에서는 이를 기본값으로 사용하는 것이 표준처럼 굳어져 있습니다.
결론적으로, 배치를 0번 차원에 두는 것은 GPU 연산 속도와 코드의 유지보수 측면에서 현대 딥러닝 모델의 표준적인 설계 방식이라고 이해하시면 됩니다.

### 단어 설명
RNN(Recurrent Neural Network)과 FC(Fully Connected Layer, 전결합층)는 데이터를 처리하는 방식과 구조에서 근본적인 차이가 있습니다.

1. Fully Connected Layer (FC)
독립적인 처리: FC는 입력 데이터 간의 관계를 시계열 순서나 흐름으로 보지 않습니다. 입력된 모든 노드가 출력 노드와 각각 연결되어, 들어오는 데이터(벡터)를 독립적으로 계산합니다.
고정된 크기: 입력과 출력의 크기가 고정되어 있으며, 정적인 데이터를 처리하는 데 적합합니다(예: 이미지 분류의 마지막 단).
2. RNN (Recurrent Neural Network)
순차적인 정보 유지: RNN은 이름처럼 **'순환'**하는 구조를 가집니다. 시퀀스 데이터(시간 순서가 있는 데이터, 예: 언어, 주가)를 처리할 때, 이전 단계에서 계산된 히든 스테이트(Hidden State) 값을 다음 단계의 입력으로 다시 활용합니다(3:56-4:18).
맥락 파악: 이전까지 입력된 단어들의 정보를 기억하고 현재 입력과 결합하기 때문에, 문장의 맥락이나 데이터의 흐름을 파악하는 데 특화되어 있습니다.
요약 비교
FC: 데이터 간의 독립성을 전제로 하며, 전체 데이터의 특징을 정적으로 추출할 때 사용합니다.
RNN: 데이터의 **'순서(Order)'**를 중요하게 생각하며, 과거의 정보를 기억해 현재의 결과에 반영합니다(1:17-1:25).
